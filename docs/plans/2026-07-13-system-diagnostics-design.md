# AI Watch 系统诊断设计

## 目标

提供一个只读系统诊断入口，让运维人员确认 CLI、SQLite、调度器和运行时目录是否处于预期状态，同时不暴露凭证、Webhook、宿主目录、配置内容或命令原始输出。

## 方案

新增独立的 `internal/diagnostics` 聚合服务，由 API 层注入现有 Scanner、Store 和 Job Manager。服务只返回白名单结构：

- Codex、Claude CLI 的可用性、二进制标签和版本。
- SQLite schema 版本、数据库文件逻辑大小、事件数和计划任务数。
- 当前活跃任务数和运行目录条目数。
- 可热更新字段与需要重启字段的静态说明。

CLI 检查先使用 `exec.LookPath`，再用两秒超时执行 `--version`。路径只返回可执行文件 basename；版本只保留第一条非空文本并限制长度。失败原因被归类为未发现、超时或版本不可读，不返回 stderr/stdout 原文。

SQLite 统计在 Store 内使用同一数据库连接和只读查询完成；文件大小来自数据库文件元数据，不扫描目录。运行目录只统计第一层条目，不返回文件名。

## API

`GET /api/diagnostics` 返回一次快照。接口不接收参数、不执行写操作，也不会触发任务、迁移、清理或通知。

## 界面

新增“系统诊断”导航页，延续现有暗色实时信号控制台。页面唯一操作是“刷新诊断”。信息按诊断总线组织为 CLI、数据层、运行时和配置生命周期四组；状态同时使用图标、文字与颜色表达。

移动端使用单列布局，刷新按钮保持至少 44px 触控高度；加载超过 300ms 显示明确反馈，错误提供重试入口，动画遵守 `prefers-reduced-motion`。

## 测试

- CLI 检查覆盖可用、缺失、超时和输出限长。
- Store 统计覆盖 schema、事件数、计划数和逻辑大小。
- API 覆盖只读响应、字段白名单和敏感字符串缺失。
- 前端执行 TypeScript 生产构建。
