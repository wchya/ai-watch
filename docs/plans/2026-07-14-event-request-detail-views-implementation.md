# 事件双视图与 CLI 请求详情实施计划

**设计：** `docs/plans/2026-07-14-event-request-detail-views-design.md`

## 阶段 1：请求事件模型

1. 补齐触发来源、客户端 IP、阶段、目标和代理元数据。
2. 采集 CLI 版本、DNS 预解析、执行时间和结果字段。
3. 保持 start/log/end 关联与统一脱敏。
4. 增加配对、错误路径和安全测试。

## 阶段 2：前端请求聚合

1. 保留原始事件类型和结构化数据。
2. 按 requestId 聚合请求开始、输出和结束事件。
3. 构建摘要统计和可展开请求详情。

## 阶段 3：事件双视图

1. 新增列表/时间线切换控件。
2. 列表视图展示事件表格和请求详情。
3. 时间线复用现有事件卡片。
4. 使用 localStorage 保存视图偏好。

## 阶段 4：Redis 刷新反馈

1. 刷新期间显示工作区加载遮罩。
2. 禁用重复刷新并保留已有数据。
3. 验证成功、失败和并发刷新行为。

## 阶段 5：验收

1. `go test ./...`
2. `npm run build`
3. `docker compose build ai-watch`
4. `docker compose up -d ai-watch`
5. 健康接口和容器状态检查。
