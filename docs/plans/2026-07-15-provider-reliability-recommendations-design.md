# Provider 可靠性诊断建议设计

**日期：** 2026-07-15  
**状态：** 已确认

可靠性 API 根据当前时间窗的请求样本，为每个 Provider 生成确定性诊断。建议等级包括 `recommended`、`healthy`、`observe`、`pause`、`insufficient`。判断使用完成样本数、成功率、P95、当前连续失败、超时/过载占比和与最佳线路的相对延迟。

规则不调用大模型，不自动切换或停用 Provider。API 返回标题、解释原因和建议动作；前端只负责展示。历史 Provider 仍可诊断，但不提供操作入口。样本不足五次时不得给出健康或暂停结论。

推荐主线路必须满足至少五个完成样本、成功率不低于 95%、当前无连续失败，并且在健康候选中排名第一。成功率低于 70%或当前连续失败至少三次时建议暂停；成功率低于 90%、存在连续失败、超时/过载占比至少 20%或 P95 高于最佳健康线路两倍时建议观察。
