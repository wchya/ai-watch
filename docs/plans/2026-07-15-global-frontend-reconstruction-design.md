# AI Watch 全局前端重构设计

**日期：** 2026-07-15  
**状态：** 已确认

## 目标

把当前由页面级硬编码颜色和局部补丁组成的界面，重构为完整的语义化设计系统。主题切换必须覆盖所有页面和组件；事件与请求日志分离；Redis Value 变成可读、可操作的数据检查器；异步刷新提供明确反馈。

## 设计方向

产品定位为面向开发者和运维人员的 AI CLI 可靠性控制台。视觉采用“信号台账”方向：信息密集但分组清晰，数据与状态优先，装饰克制。

### 字体

- 标题与正文：`Fira Sans` / `Noto Sans SC`。
- 数据、时间、Key、请求 ID 与代码：`Fira Code`。
- 正文字号不低于 12px，移动端输入控件不低于 16px。

### 布局

- 8px 间距基线；4px 用于紧凑数据标签。
- 页面最大宽度、导航宽度、面板圆角、阴影和分隔线全部使用 Token。
- 数据表优先纵向浏览；移动端转卡片，不依赖水平滚动完成主要操作。

### 标志性元素

使用“信号轨道”作为唯一视觉签名：状态指示、请求生命周期和数据层级使用细线与节点表达。其他区域保持平静，避免大量发光边框。

## 语义 Token

建立以下类别，并禁止业务组件继续新增裸色值：

- 页面：`--app-bg`、`--app-bg-elevated`、`--app-grid`。
- 表面：`--surface-1`、`--surface-2`、`--surface-3`、`--surface-hover`、`--surface-active`。
- 文字：`--text-primary`、`--text-secondary`、`--text-muted`、`--text-inverse`。
- 边界：`--line-subtle`、`--line-default`、`--line-strong`、`--focus-ring`。
- 状态：`--status-info`、`--status-success`、`--status-warning`、`--status-danger` 及对应弱背景。
- 控件：输入框、按钮、代码区、弹层和遮罩专用 Token。
- 尺寸：间距、圆角、阴影、动画时长和 z-index。

## 三套主题

### 深海终端

深蓝黑背景、青色主信号、低亮度蓝灰分隔线。适合长时间监控。

### 石墨信号

中性石墨背景、薄荷绿主信号、减少蓝色偏移。适合低饱和工作环境。

### 极昼控制台

浅灰蓝页面、白色表面、深蓝文字、青蓝主信号。所有固定暗色代码区、表单、弹层和边框必须单独映射，不能使用全局滤镜。

主题入口固定放在顶栏，使用带名称和色板的选择器；设置页保留完整预览和说明。选择即时预览，保存到 Redis；失败时保留当前视觉并明确提示未持久化。

## 事件记录与请求日志

事件页面拆成两个数据页签：

- **事件记录**：任务、调度、配置、清理、恢复和系统生命周期事件。
- **请求日志**：以 `requestId` 为一行，聚合 `request_start`、`request_log` 和 `request_end`。

两个页签共享筛选入口，但请求日志增加状态、CLI、Provider、触发来源和时间筛选。请求详情展示当前能够可靠获得的全部脱敏信息：

- IP、任务、Provider、CLI、CLI 版本、模型、配置来源。
- 请求时间、结束时间、耗时、触发来源、阶段和尝试次数。
- 目标 URL、主机、端口、DNS 预解析、代理模式和脱敏代理端点。
- 请求体只展示安全摘要：CLI 参数名、模型、Prompt 字节数和哈希，不返回 Prompt 原文。
- 返回状态、退出码、分类、响应日志片段、截断状态。
- 错误类型、脱敏错误和下一次运行时间。

普通事件保留列表/时间线切换；请求日志使用列表和展开详情，不混入普通事件时间线。

## Redis 数据检查器

- 刷新期间显示覆盖工作区的 loading、按钮 loading 和可访问状态文字；保留旧内容避免布局跳动。
- String：结构预览与原文编辑分离；JSON 使用树形节点、层级线、类型标签和折叠。
- Hash/List/Set/ZSet：使用统一集合表格，表头、值、类型和操作列稳定对齐。
- Key、TTL、版本、内存、截断和只读状态使用紧凑元数据栏。
- 避免每个值单元格都有亮白边框；通过表面层级、行分隔和悬停状态建立结构。

## 无障碍与响应式

- 正文对比度至少 4.5:1；状态不只依赖颜色。
- 所有交互目标至少 44px，支持键盘、焦点环和 Escape。
- 动画 150–240ms，并遵守 `prefers-reduced-motion`。
- 验证 375px、768px、1024px、1440px；移动端请求详情和 Redis 集合转为卡片/纵向详情。

## 验收

- 三套主题覆盖导航、页面、面板、表单、表格、代码区、弹层和状态组件。
- 主题切换入口在顶栏可发现，设置持久化成功。
- 不再通过 `filter` 实现主题。
- 事件和请求列表完全分开；请求记录按 requestId 聚合。
- Redis 刷新 loading 可见，成功和失败都会结束。
- Redis Value 在三套主题和小屏下可读可操作。
- 运行 Go 全量测试、前端构建、Docker 构建和健康检查。
