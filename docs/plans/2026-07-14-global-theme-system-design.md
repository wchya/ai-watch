# 全局三主题系统设计

**日期：** 2026-07-14  
**状态：** 已确认

## 目标

为 AI Watch 提供三套可手动切换的全局主题。主题选择写入 Redis Settings，所有访问同一实例的浏览器共享当前主题。

主题系统必须覆盖现有总览、供应商、计划任务、事件、系统诊断、设置、Redis 管理、抽屉、确认框、Toast 和移动端布局。

## 架构

采用 CSS Token 与根节点 `data-theme` 属性。React 组件不为每套主题复制结构，也不动态加载独立样式表。

主题 ID：

- `deep-ocean`
- `graphite-signal`
- `arctic-daylight`

当前主题作为 UI 设置字段写入现有 Redis Settings。后端校验主题 ID，只接受上述三个值。旧设置缺少该字段时使用 `deep-ocean`。

前端启动后读取 Settings，并在应用根节点设置 `data-theme`。切换时先应用候选主题，再调用 Settings API 持久化；保存失败则恢复原主题并显示错误提示。

浅色主题独立定义面板、边框、阴影、代码区、输入框、状态色和危险操作色，保证普通正文至少 4.5:1 对比度。

## 切换交互

顶部栏新增主题按钮。点击后显示三张主题预览卡，每张卡包含：

- 主题名称和明暗模式。
- 背景、面板、文字与强调色预览。
- 小型界面缩略结构。
- 当前主题标记。

预览面板支持点击外部关闭、Escape 关闭、键盘 Tab 顺序和明确焦点状态。主题过渡控制在 180–240ms，并遵守 `prefers-reduced-motion`。

## CSS Token

现有全局颜色逐步收敛为语义变量：

- 页面背景、面板、面板增强层。
- 主要文字、次要文字、弱化文字。
- 边框、输入框、悬停层。
- 主强调、成功、警告、危险和紫色辅助色。
- 代码背景、遮罩、阴影与环境光。

本次优先覆盖所有公共结构和 Redis 管理页，不通过组件内硬编码主题判断切换颜色。

## 错误处理

- 后端拒绝未知主题 ID。
- 保存失败时前端恢复旧主题。
- Redis 不可用时保留当前视觉状态并提示未持久化。
- 旧数据自动使用默认主题，不要求迁移。

## 验证

- 三套主题覆盖所有主要页面和浮层。
- 主题选择刷新后从 Redis 恢复。
- 两个浏览器访问同一实例时共享主题。
- 保存失败时正确回滚。
- 验证键盘、焦点、Escape 和减少动画设置。
