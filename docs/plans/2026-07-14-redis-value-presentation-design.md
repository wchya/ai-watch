# Redis Value 展示优化设计

**日期：** 2026-07-14  
**状态：** 已确认

## 目标

提升 Redis 管理页中 Value 的可读性和操作体验。所有可以解析为 JSON 的值都自动使用格式化、语法高亮和可折叠的结构化视图展示；Hash、List、Set、ZSet 统一改为表格布局。

## JSON 展示

- String 整体以及集合中的每个 Value 或 Member 都尝试解析 JSON。
- 合法 JSON 默认展示紧凑结构预览，可展开完整层级。
- Object、Array、字符串、数字、布尔值和 null 使用不同语义色。
- Object 和 Array 支持按层级折叠，并显示字段或元素数量。
- 长文本和大型 JSON 默认截断，避免撑坏页面。
- 提供复制原文、展开全部和收起全部操作。
- 非 JSON 值使用统一的代码文本预览。

## String 编辑器

String 使用“预览 / 编辑”双模式。预览模式优先显示 JSON Viewer；编辑模式保留原始文本编辑器。

合法 JSON 可以一键格式化。保存前不强制要求 String 必须是 JSON，但当内容看起来像 JSON 且解析失败时显示明确提示。现有保存确认、版本冲突与自动预热逻辑保持不变。

## 集合表格

- Hash：Field、Value、类型、操作。
- List：Index、Value、类型、操作。
- Set：Member、类型、操作。
- ZSet：Member、Score、类型、操作。

表格 Value 单元格展示紧凑预览。合法 JSON 可在行内展开完整 Viewer；非 JSON 长文本截断并支持查看完整内容。编辑操作继续使用现有确认流程。

## 响应式与视觉

表格沿用 Redis 控制台的暗色运维风格，表头保持固定，数据区域独立滚动。JSON Viewer 使用等宽字体、缩进引导线和低饱和语义色，不引入新的前端依赖。

窄屏保留表格语义，通过局部横向滚动避免列被隐藏；Value 列获得主要宽度，操作列保持可触达。

## 验证

- 验证 String JSON 与普通文本的预览、编辑和保存。
- 验证 Hash、List、Set、ZSet 中 JSON 与普通值混合展示。
- 验证折叠、展开、复制、长文本截断和空集合。
- 验证桌面与移动端布局。
