# 定时可靠性摘要设计

## 目标

每天按用户配置的时间与时区，通过现有钉钉通知渠道发送一次可靠性摘要。摘要只基于脱敏请求事件，展示整体成功率、P95 延迟、数据覆盖情况和 Provider 建议，不包含凭证、Prompt、完整响应或原始 CLI 输出。

## 配置

- 启用或关闭自动摘要。
- 发送时间，精确到分钟。
- 时区，首期提供常用 IANA 时区选项。
- 统计范围：24 小时、7 天或 30 天。
- 手动预览和手动发送始终可用；手动发送不占用当天自动发送名额。

配置沿用全局设置的持久化边界，通过现有设置 API 保存。钉钉 Webhook 仍由已有加密配置管理，摘要功能不读取或返回明文。

## 调度与幂等

Manager 启动一个轻量定时循环，每 30 秒检查一次本地时区时间。到达当天配置时间后：

1. 查询结构化事件，确认当天是否已有 `source=scheduled` 的成功事件。
2. 生成可靠性摘要并发送。
3. 成功后记录 `reliability_digest_sent`；内存同时缓存已发送日期，降低重复查询。
4. 失败时记录 `reliability_digest_failed`，不写成功标记，后续检查周期自动重试。

进程重启后仍通过持久化成功事件保证同一自然日只自动发送一次。手动发送记录 `source=manual`，不会阻止自动发送。

## API 与界面

- `GET /api/reliability/digest/preview`：生成 Markdown 预览，不发送消息。
- `POST /api/reliability/digest/send`：生成并立即发送，未配置钉钉时返回明确冲突错误。
- 设置页新增“定时可靠性摘要”卡片，包含开关、时间、时区、范围、预览和发送按钮。
- 预览以主题化代码块展示；发送、预览、保存均有独立 loading，避免重复提交。

## 验证

- 设置 schema v11 往返保存。
- 预览不会调用通知器。
- 手动发送成功且不写 scheduled 日期。
- 自动发送同一天只成功一次，失败后可以重试。
- API 错误码、前端设置保存、浅色主题与移动布局均有覆盖。
