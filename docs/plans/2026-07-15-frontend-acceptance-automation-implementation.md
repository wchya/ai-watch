# AI Watch 前端验收自动化实施计划

**设计：** `docs/plans/2026-07-15-frontend-acceptance-automation-design.md`

## 阶段 1：测试运行基础

1. 引入 Playwright Test 和 Chromium 浏览器运行配置。
2. 增加 `test:e2e` 与浏览器安装命令。
3. 配置 Vite webServer、trace、截图和失败产物目录。

## 阶段 2：稳定 API 场景

1. 建立共享 API Mock，覆盖应用初始化接口。
2. 提供供应商、Redis 和设置场景数据。
3. 记录主题保存请求、未匹配请求和浏览器控制台错误。
4. 支持 Redis 延迟刷新和失败后恢复。

## 阶段 3：浏览器验收用例

1. 验证顶栏三主题切换和持久化请求。
2. 验证 Redis 刷新期间 loading、成功结果和失败恢复。
3. 验证 320px/375px 下供应商操作区、导航和编辑弹层。
4. 验证 320/375/768/1024/1440px 无非预期横向溢出。
5. 为三主题关键页面保存视觉证据。

## 阶段 4：文档与门禁

1. README 增加浏览器验收命令与产物说明。
2. 更新项目进度文档，记录自动化覆盖和剩余人工检查。
3. 运行前端构建、Playwright、Go 测试和 Compose 配置检查。
4. 只修复由真实浏览器证据证明的布局或行为问题。
