# AI Watch Provider 可靠性告警实施计划

**设计：** `docs/plans/2026-07-15-provider-reliability-alerts-design.md`

## 阶段 1：设置与存储

1. 扩展 Settings、默认值和校验。
2. 增加 SQLite v9 迁移并更新 Redis/SQLite 设置读写。
3. 补默认值、迁移和热更新测试。

## 阶段 2：告警状态机

1. 在请求事件持久化后异步评估 24 小时可靠性。
2. 实现阈值、多原因合并、冷却和恢复计数。
3. 通过结构化事件恢复状态并记录触发、抑制、恢复和发送失败。
4. 复用钉钉消息发送能力，确保失败不影响任务。

## 阶段 3：前端

1. 增加设置类型和保存字段。
2. 在“设置与通知”增加可靠性告警卡。
3. 在可靠性页面展示当前阈值状态。

## 阶段 4：验证

1. 补状态机、API、设置存储和隐私测试。
2. 扩展 Playwright 设置保存与移动端验收。
3. 更新 README 和项目进度。
4. 运行全量门禁。
