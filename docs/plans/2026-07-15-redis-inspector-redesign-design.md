# Redis Inspector 界面重构设计

## 目标

将 Redis 管理从按钮和边框密集的后台表格，重构为以 Value 内容为主体的轻量数据检查器。保留现有写入、版本冲突保护、危险操作确认和应用缓存预热能力。

## 交互原则

- 普通字符串、数字和布尔文本均按纯文本显示，不展示 `JSON Number`、`Number` 等类型标签。
- 只有解析结果为 JSON Object 或 JSON Array 时，才提供格式化树和原文切换。
- 编辑、删除等行操作在悬停、键盘聚焦或触摸设备上显示，降低常驻视觉噪音。
- 集合新增表单默认收起；TTL、重命名和删除收进 Key 设置入口。
- Value 内容保持最高视觉权重，边框、工具条和复制按钮使用低对比度样式。
- Escape 可退出行内编辑；危险操作继续要求输入完整 Key 确认。

## 页面结构

1. Key 检查器顶部显示类型、Key 名和轻量复制、刷新、设置操作。
2. TTL、大小、内存、编码和版本指纹以紧凑元信息带展示。
3. String 使用统一 Value 画布；JSON 才显示格式化视图。
4. Hash、List、Set、ZSet 移除低价值的类型列和固定表头，操作区按上下文显示。
5. JSON 树保留逐级展开能力，全部展开/收起工具条弱化并在交互时增强。

## 响应式与可访问性

- 无 hover 的设备始终显示必要行操作。
- 所有图标按钮提供 `aria-label` 和可见焦点。
- 窄屏压缩元信息和字段列宽，保留 Value 的主要阅读空间。
- 遵循系统减少动画设置。

## 验证

- TypeScript/Vite 构建。
- String、JSON Object/Array、Hash、List、Set、ZSet 的查看与编辑回归。
- 鼠标、键盘和触摸设备下操作可发现性检查。
- 三套主题及移动端布局检查。
