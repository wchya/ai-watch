# 事故复盘报告设计

## 目标

把事故中心已有的脱敏事实固化为可编辑、可交付的复盘报告。报告首次生成时创建快照，之后不受事件保留策略清理影响。

## 数据模型

每个事故最多对应一份复盘：

- 状态：`draft`、`completed`；
- 自动快照：事故摘要、影响对象、开始/恢复时间、持续时间、失败分类、关联 Job/Request、关键时间线和恢复方式；
- 人工字段：根因、处置总结、后续行动、负责人；
- 后续行动包含内容、负责人、是否完成；
- 创建、更新时间和完成时间。

自动快照在首次生成后只读。人工字段可以保存；完成后不可编辑，重新打开后恢复编辑。

## API

- `GET /api/incidents/:id/postmortem`：读取已有复盘；不存在返回 404；
- `POST /api/incidents/:id/postmortem`：基于当前事故生成首份快照，重复调用幂等返回现有复盘；
- `PUT /api/incidents/:id/postmortem`：保存人工字段；
- `POST /api/incidents/:id/postmortem/complete`：标记完成；
- `POST /api/incidents/:id/postmortem/reopen`：重新打开；
- `GET /api/incidents/:id/postmortem/markdown`：返回可下载 Markdown。

生成、保存、完成和重开均写结构化审计事件。Markdown 只包含脱敏快照和人工字段，不读取 Prompt、凭证、Webhook、代理明文或完整供应商输出。

## 页面

事故详情增加“生成复盘”或“查看复盘”入口。复盘使用侧边抽屉：

- 顶部展示状态、事故主体和持续时间；
- 自动证据区只读；
- 人工结论区可编辑根因、处置总结、负责人和后续行动；
- 支持保存草稿、标记完成、重新打开、复制 Markdown 和下载 `.md`；
- 写操作提供 loading、全局幂等和 2.5 秒刷新。

页面复用语义主题变量，极昼主题不出现固定暗色；移动端改为全宽抽屉且无横向溢出。

## 错误处理与验证

- 事故不存在返回 404；
- 已完成复盘直接更新返回 409；
- 文本和行动项数量、长度受限；
- 复盘 Store 覆盖 SQLite 与 Redis 往返；
- API 测试覆盖快照稳定性、保存、完成、重开、Markdown 脱敏和事件清理后仍可读；
- Playwright 覆盖生成、编辑、完成、复制/下载入口、极昼主题和移动端。
