# AI Watch 全局前端重构实施计划

**设计：** `docs/plans/2026-07-15-global-frontend-reconstruction-design.md`

## 阶段 1：设计系统基础

1. 建立完整语义 Token 和三套主题映射。
2. 替换全局页面、导航、面板、按钮、输入框、表格、代码区与弹层基础样式。
3. 删除主题 `filter` 和高亮白边补丁。
4. 增加顶栏主题选择器、即时预览、保存失败反馈和键盘交互。

## 阶段 2：事件与请求拆分

1. 后端普通事件持久化结构化 Data，但不把完整输出写入长期事件流。
2. 增加按任务批量读取请求事件的 API 或聚合接口。
3. 普通事件列表保留列表/时间线切换。
4. 请求日志独立列表，按 requestId 聚合并支持展开详情。
5. 补请求体安全摘要、错误、返回日志和下一次运行时间。

## 阶段 3：Redis 数据检查器

1. 统一 Redis 页面加载、错误、空状态和成功反馈。
2. 重写 JSON 树形查看器。
3. 重写 String 预览/编辑和集合表格。
4. 统一 Key 元数据、TTL、危险操作和响应式布局。

## 阶段 4：全局页面迁移

1. 总览、供应商、计划、诊断、设置和任务详情迁移到语义 Token。
2. 清理固定白边、固定暗色和重复媒体查询。
3. 验证三主题的文字、图标、状态和焦点对比度。

## 阶段 5：验证与部署

1. `go test ./...`
2. `npm run build`
3. `docker compose build ai-watch`
4. `docker compose up -d ai-watch`
5. 检查健康接口、375/768/1024/1440px 和三套主题。
