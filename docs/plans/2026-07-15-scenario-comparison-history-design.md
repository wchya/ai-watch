# 场景对比历史设计

## 目标

为多 Provider 场景对比提供可恢复、可筛选、可深链接的历史入口，并支持按原场景和 Provider 集合重新运行。

## 数据来源

- 列表和详情都从 `scenario_comparison_started` 结构化事件恢复。
- 活动 Job 优先提供实时状态；Job 回收后使用 `request_end` 恢复 Request ID、耗时、错误类型和脱敏摘要。
- 不复制场景 Prompt、期望文本、密钥或完整 CLI 输出。

## 状态

- `running`：至少一条线路仍在排队或运行。
- `completed`：所有线路成功。
- `partial_failed`：批次已结束且至少一条线路失败或启动失败。
- 事件超过保留策略后返回历史不存在/已过期提示。

## 页面

- 新增 `/comparisons` 一级路由和侧边栏入口。
- 支持全部、运行中、已完成、含失败筛选。
- 左侧批次列表，右侧详情排名；移动端纵向排列。
- Request ID 打开统一请求详情。
- 重跑前确认，使用原场景、CLI 和 Provider 集合创建新批次。

## 幂等与安全

- 重跑使用全局 Idempotency-Key。
- 服务端重新验证场景仍启用且 CLI 兼容。
- 不修改 Provider、计划、故障切换组或宿主机配置。

## 验收

- 列表和详情状态一致。
- 运行批次自动刷新，完成后停止轮询。
- 历史详情支持浏览器刷新和前进后退。
- 原集合重跑返回新的稳定批次 ID。
